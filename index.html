How It Works — Web Audio API Architecture 
┌──────────────────────────────────────────────────────────────────────────┐
│                     AudioRouter Signal Flow (v2)                         │
├──────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  SOURCE INPUT (two modes):                                               │
│    ┌─ Local File ─── File API → URL.createObjectURL(file) ─────────┐    │
│    └─ Stream URL ── URL validation → crossOrigin="anonymous" ───────┘    │
│                            │                                             │
│                            ▼                                             │
│  <audio crossOrigin="anonymous">  (main hidden audio element)           │
│       │   .src = blobURL  OR  remoteURL                                  │
│       ▼                                                                  │
│  createMediaElementSource(audioEl)  ← single AudioContext source        │
│       │                                                                  │
│       ├──► GainNode (master volume)                                      │
│       │         │                                                        │
│       │         ▼                                                        │
│       │   AnalyserNode ──► canvas visualiser                            │
│       │         │                                                        │
│       │         ▼                                                        │
│       │   AudioContext.destination  (system default output)             │
│       │                                                                  │
│       └── [For each checked Bluetooth device:]                          │
│                 │                                                        │
│                 ▼                                                        │
│           DelayNode (0–500 ms, per-device latency calibration)          │
│                 │                                                        │
│                 ▼                                                        │
│           MediaStreamAudioDestinationNode                               │
│                 │                                                        │
│                 ▼                                                        │
│           new HTMLAudioElement()                                         │
│                 │ .setSinkId(deviceId)   ← routes to BT device          │
│                 │ .srcObject = stream                                    │
│                 └──► 🔊 Bluetooth Speaker / Headphones                  │
│                                                                          │
└──────────────────────────────────────────────────────────────────────────┘

Each checked device gets its own independent routing chain so latency offsets never affect other routes.

⚠ Local files: URL.createObjectURL(file) creates a temporary blob URL (blob://…) that is valid for the lifetime of the page. crossOrigin is NOT set for local files — it is only applied for remote URLs.

⚠ Browser support: setSinkId() is supported in Chromium-based browsers (Chrome, Edge, Opera). Firefox and Safari do not support it yet. Run this app from a local file (file://) or a trusted HTTPS origin.
